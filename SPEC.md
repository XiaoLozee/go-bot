# Go-bot 规格说明（SPEC）

> 状态：当前实现 + 最近规划收口版  
> 适用版本：新宿主架构 v1 持续演进中

---

## 1. 文档目标

本文档定义 Go-bot 当前阶段的：

- 产品定位
- 功能边界
- 模块能力
- 数据存储策略
- 非功能要求

它回答的问题是：

> **Go-bot 现在是什么、必须做什么、暂时不做什么。**

---

## 2. 产品定位

Go-bot 是一个 **AI First 的自托管机器人宿主平台**。

核心定义：

- **AI 是内建核心能力**
- **插件是第一扩展系统**
- **连接、配置、存储、运维统一由宿主管理**
- **WebUI 是统一控制台**

一句话说明：

> Go-bot = AI 内核 + 插件生态 + 多连接宿主 + 管理后台

---

## 3. 当前目标用户

当前主要面向：

- 个人开发者
- 小团队自托管部署者
- 需要长期运行 QQ / OneBot Bot 的维护者

部署目标优先级：

1. Linux 单机
2. Windows 开发环境兼容

---

## 4. 当前范围

## 4.1 已在范围内

### 宿主能力

- 程序入口 `main.go`
- 统一 runtime
- 多连接管理
- Admin API
- WebUI 后台
- 配置加载、校验、保存、备份

### 协议 / 接入

- OneBot v11
- NapCat HTTP Action
- `ws_server`
- `ws_reverse`
- `http_callback`

### 插件体系

- `external_exec` 为主线
- 外部插件安装 / 升级 / 卸载 / 启停 / 重载
- builtin 仅保留极少量宿主级测试能力

### AI 核心

- 回复策略
- 群策略
- 视觉识别接入
- 候选记忆 / 长期记忆
- 群画像 / 成员画像 / 关系边
- 轻量离线反思
- 群级观察中心
- 聊天记录存储（文本 + 图片）
- 私聊人格模板

### 存储

- SQLite
- MySQL
- PostgreSQL
- 媒体存储：
  - local
  - Cloudflare R2

## 4.2 当前不在范围内

- 多租户
- SaaS 化
- 分布式 Worker 集群
- 插件市场
- 视频 / 语音纳入 AI 主消息模型
- 私聊人格自动孵化
- 按用户自动切换私聊人格

---

## 5. 功能规格

## 5.1 运行时与连接

系统必须支持：

- 启动 / 关闭宿主
- 维护多个连接
- 独立查看连接状态
- 独立探活
- 独立保存连接配置
- 允许没有启用连接时仍可进入后台

连接管理后台必须支持：

- 查看总数 / 在线数 / 启用数
- 按类型区分
- 查看单连接详情
- 新建 / 编辑 / 删除连接

---

## 5.2 插件系统

### 主路线

插件系统当前以 `external_exec` 为标准扩展方式。

插件必须支持：

- 安装
- 升级 / 覆盖安装
- 启用 / 停用
- 重载
- 恢复
- 卸载
- 结构化配置

### 插件类型

#### builtin

- 极少量宿主测试能力
- 允许开关
- 不作为业务功能主线

#### external_exec

- 正式业务插件主线
- 支持 Linux 友好的打包与部署

### 插件与 AI 的关系

插件可以：

- 提供工具能力
- 提供上下文能力
- 提供独立工具能力

但插件不应：

- 直接接管 AI 主人格
- 直接接管 AI 最终回复权

---

## 5.3 Admin API

Admin API 必须承担：

- runtime 快照
- 配置管理
- 连接管理
- 插件管理
- AI 观察与配置管理
- 后台鉴权
- WebUI bootstrap

API 风格要求：

- 以管理台使用为第一目标
- 返回结构稳定
- 对 WebUI 友好
- 尽量使用结构化 JSON 返回

---

## 5.4 WebUI

WebUI 必须提供统一控制台，而不是碎片化页面集合。

当前核心页面：

- 基础信息
- AI 功能
- 网络配置
- 插件管理
- 系统配置
- 操作审计

设计要求：

- 中文产品化表达
- 结构化表单优先
- 多主题浅色系切换
- 更像正式控制台，不像个人工具页

---

## 6. AI 规格

## 6.1 AI 定位

AI 是 Go-bot 的核心能力，不是插件。

AI 必须承担：

- 是否回复
- 如何组织上下文
- 如何使用记忆
- 如何使用群理解结果
- 如何使用视觉摘要
- 私聊使用哪套当前人格模板

---

## 6.2 群聊 AI

群聊 AI 是当前主线。

群聊 AI 当前应支持：

- 群聊开关
- 是否要求 @
- 问句触发
- 冷却时间
- 上下文条数
- 输出 Token 限制
- 群级策略覆盖

群聊 AI 当前允许进行：

- 群画像收敛
- 成员画像收敛
- 关系边收敛
- 群梗候选沉淀
- 候选记忆整理

---

## 6.3 私聊 AI

私聊 AI 当前不是“学习型人格孵化”路线，而是“模板型人格切换”路线。

### 私聊人格规则

- 可以保存多套人格模板
- 同时仅一套人格全局生效
- 私聊回复固定读取当前生效人格
- 私聊消息不自动回写人格模板

### 私聊人格模板字段

- `id`
- `name`
- `description`
- `bot_name`
- `system_prompt`
- `style_tags`
- `enabled`

---

## 6.4 AI 聊天数据存储

当前主消息模型聚焦：

- 文字
- 图片

### 不纳入 v1 主模型

- 视频
- 语音

### 文本策略

- 全文留存

### 图片策略

- 存媒体引用
- 存视觉摘要
- 后台可直接看图
- 不依赖 QQ 原始 URL

### 消息主记录必须至少包含

- `message_id`
- `connection_id`
- `chat_type`
- `group_id`
- `user_id`
- `sender_role`
- `sender_name`
- `reply_to_message_id`
- `text_content`
- `has_text`
- `has_image`
- `message_status`
- `occurred_at`

### 图片记录必须至少包含

- `message_id`
- `segment_index`
- `origin_ref`
- `vision_summary`
- `vision_status`
- 关联媒体资产信息
- `preview_url`

---

## 6.5 AI 记忆与反思

当前记忆体系包含：

- 会话状态
- 候选记忆
- 长期记忆
- 群画像
- 成员画像
- 关系边
- 群观察摘要

### 反思主线

当前采用：

- 规则优先
- 后续可扩展 LLM 辅助

反思目标：

- 整理候选记忆
- 合并重复候选
- 识别冲突候选
- 生成群阶段摘要
- 收敛群画像
- 沉淀群梗候选

---

## 7. 数据与存储规格

## 7.1 配置存储

配置必须支持：

- 加载
- 校验
- 保存
- 备份
- 脱敏查看

## 7.2 结构化存储

结构化主数据支持：

- SQLite
- MySQL
- PostgreSQL

## 7.3 媒体存储

媒体支持：

- local
- R2

聊天层不直接依赖外部临时 URL，而应依赖宿主自己的媒体资产能力。

---

## 8. 安全规格

后台必须支持：

- 首次设置密码
- 登录
- 退出登录
- 登录后修改密码

安全底线：

- 不提交真实密钥
- 配置输出必须脱敏
- 后台敏感操作要有清晰反馈

---

## 9. 非功能要求

## 9.1 可维护性

- 模块职责清晰
- 旧体系逐步清理
- 文档与实现同步更新

## 9.2 可观测性

- runtime 快照可读
- 连接状态可看
- 插件状态可看
- AI 观察中心可看
- 审计记录可查

## 9.3 可扩展性

- 可增加新连接类型
- 可增加新 external_exec 插件
- 可增强 AI 存储与观察模型
- 可继续扩展 WebUI

## 9.4 兼容性

- Windows 适合开发与调试
- Linux 为生产优先

---

## 10. 成功标准

当前阶段判断项目是否达标，至少看：

1. 宿主可稳定启动
2. 多连接可管理
3. external_exec 插件可热插拔
4. WebUI 可作为统一控制台使用
5. AI 能稳定处理群聊主线
6. AI 聊天记录可查看文字与图片
7. 私聊人格可多模板保存与切换
8. 测试与构建可通过
