# Go-bot AI 内核升级方案 v2

> 状态：已完成第二轮落地（持续迭代中）  
> 更新日期：2026-04-20

---

## 1. 升级目标

本轮升级目标不是做一个“大而全”的 Agent，而是把当前 AI 内核升级成一个**更懂群、更可控、更便于运营和观察**的产品级能力层。

核心原则：

- **KISS**：先把在线回复、记忆、画像、群策略、后台整理做稳，再考虑更复杂的离线智能体。
- **YAGNI**：暂不引入重型调度系统、消息队列或复杂多 Agent 编排。
- **DRY**：统一通过 AI 核心拼装 Prompt、统一通过 Store 写入状态。
- **SOLID**：回复门控、画像提取、视觉识别、群策略、后台整理分别拆分，职责单一。

---

## 2. 本轮已完成能力

### 2.1 在线回复链路

已具备：

- AI 服务商配置与热更新
- 群聊 / 私聊回复门控
- @ 触发、问句触发、冷却时间
- 图片理解接入（收到图片 → 视觉模型 → 参与回复）
- Prompt 规划增强，避免回复像“客服问答”

### 2.2 记忆与画像

已具备：

- 会话窗口
- 候选记忆池
- 长期记忆
- 群画像 `group_profile`
- 成员画像 `user_in_group_profile`
- 关系边 `relation_edge`
- SQLite / MySQL / PostgreSQL 持久化

### 2.3 群级 AI 策略

已新增群级 AI 策略 `group_policies`，支持按群覆盖：

- 是否允许回复
- 是否必须 @
- 是否允许问句触发
- 冷却时间
- 最大上下文条数
- 最大输出 token
- 是否允许视觉识别
- 群附加 Prompt

设计约束：

- **删除群策略 = 继承全局**
- `reply_enabled=false` 表示该群只学习、不主动回复
- `vision_enabled` 采用**收窄策略**：全局视觉关闭时，群策略不能强开

### 2.4 WebUI / 调试观测

已具备：

- AI 配置表单化编辑
- 服务商、视觉模型、记忆策略分区配置
- 群策略结构化编辑
- 群画像 / 成员画像 / 关系边可视化观察
- 候选记忆 / 长期记忆管理
- AI 运行状态与最近错误展示

### 2.5 轻量后台整理（Reflection Lite）

已新增第一版后台整理能力，不依赖额外调度系统：

- 周期性扫描候选记忆
- 支持在 Admin API / WebUI 中手动触发“立即整理记忆”
- 达到晋升阈值时自动晋升到长期记忆
- 清理过期候选记忆
- 清理过期长期记忆
- 基于 `raw_message_log` 回放最近群消息，后台收敛更新群画像
- 为群画像生成可读的阶段摘要
- 将高频群梗沉淀为 `group_meme` 候选记忆
- 在 Snapshot 中暴露整理运行状态、最近整理时间与结果摘要

这一步属于 **Reflection Worker 的轻量版本**，目标是先把“候选晋升 / 过期清理”做稳定。

## 3. 当前架构

在线链路：

1. 接收消息
2. 提取文本 / 图片识别结果
3. 更新会话窗口
4. 捕获候选记忆 / 长期记忆 / 群画像 / 成员画像 / 关系边
5. 应用群策略
6. Reply Gate 判断是否回复
7. Reply Plan 调整语气 / 长度 / 目标
8. Prompt 拼装并调用模型生成
9. 写回会话与原始消息日志

后台整理链路：

1. 周期扫描候选记忆
2. 按晋升阈值自动晋升长期记忆
3. 按 TTL 清理候选记忆
4. 按 TTL 清理长期记忆
5. 回放最近群消息并重新收敛群画像
6. 生成群阶段摘要并沉淀高频群梗候选
7. 更新运行摘要，供 WebUI 展示

## 4. 本轮收益

### 4.1 KISS

- 没有额外引入复杂的 Worker 基础设施
- 群策略采用完整条目配置，前后端实现更直接
- 后台整理先解决最核心的晋升和清理问题

### 4.2 YAGNI

- 暂不做复杂“群人格自动融合”
- 暂不做 LLM 驱动的离线总结任务队列
- 暂不做跨群共享人格与关系迁移

### 4.3 DRY

- 群策略统一在 AI 核心入口生效
- 视觉能力统一在图片识别入口生效
- 记忆、画像、关系统一走 Store 抽象层

### 4.4 SOLID

- `group_policy.go`：负责群级策略合并
- `vision.go`：负责图片识别与视觉注入
- `profile.go`：负责画像与关系边提取
- `planner.go`：负责轻量回复规划
- `service.go`：负责编排与运行时状态

## 6. 仍待继续的点

本轮虽然完成了第二阶段主体，但距离 PRD 仍有几个明显缺口：

1. **真正的离线反思 Worker**
   - 目前只有轻量后台整理
   - 还没有基于原始消息做群风格总结、梗沉淀、冲突消解

2. **关系图谱进一步产品化**
   - 当前已经能展示群画像和关系边
   - 后续可以增加更直观的图谱视图和筛选能力

3. **更细粒度的群策略与运营能力**
   - 如群级白名单 / 黑名单 / 仅主人可操作策略模板等

4. **非结构化消息记忆进一步增强**
   - 当前已支持媒体存储与图片识别参与回复
   - 后续可继续补齐图片 / 语音 / 视频的长期索引与检索

## 7. 下一步建议

建议下一轮继续按以下顺序推进：

### Phase E：离线反思增强

- 基于 `raw_message_log` 做群风格阶段性总结
- 沉淀群梗热词与软规则
- 对候选记忆做冲突合并与置信度调整

### Phase F：图谱产品化

- WebUI 增加群级知识图谱视图
- 支持按群筛选成员、关系边和热点主题

### Phase G：多模态记忆增强

- 为图片 / 语音 / 视频增加结构化索引
- 支持从本地存储或 Cloudflare R2 回溯媒体上下文

## 8. 结论

这轮升级已经把 Go-bot 的 AI 从“单次聊天能力”推进到“具备群级策略、群画像、关系线索、视觉参与、后台整理”的阶段。

它还不是 PRD 里的最终形态，但已经具备了继续向**更懂群的社交型 AI 内核**演进的稳定基础。
